# 更新日志

# v1.0

v1.0.0: 增加了归档功能

1、增加了归档功能，可按部门进行每月归档 默认关闭，需要在工时设置中进行开启。

2、优化了日报填报后的状态显示问题。

3、修复一些其他bug。

# v0.10

1、新增了日志审批功能。

2、优化了项目填报后，强制列表刷新的问题。

3、优化了项目日报、项目月报中，超级管理员看不到所有项目的情况。

4、优化了部分日报详情界面

5、优化了部分菜单位置和数据

  • 之前版本的用户不更新也可以

  • 如果想要更新可以单独导入附件的 menu数据，使用方法: 第1步，备份数据库中 sys_menu 表。 第2步，导入附件中的sys_menu.sql文件。 第3步：根据新导入的菜单，重新调整角色权限。

  • 或者根据demo环境手工进行调整。

6、细化了部分业务的菜单权限 老版本更新后对部分业务的操作权限需要重新配置。

# v0.9

1、新增了是否记录0工时的项目记录，可通过“应用设置” 进行设置。

2、优化了日报的查看界面。

3、修正优化了审核业务。

已审核通过的无法再次审核，无法进行编辑。

4、优化了部分前端查看页面。

5、其他bug。

# v0.7

1、新增了我的项目的功能。

2、修复了项目月报筛选项可清空问题。

3、修复了项目月报中由看不到某些特殊项目的情况。

4、增加了我的统计模块 用于查看自己的填报率。

5、增加了部门统计模块，用于查看部门人员的填报率。

6、修复了部分模块的权限问题。

7、修复了填报工时时可填报负值的问题。

8、一些其他优化。

# v0.6

1、新增了项目月报功能，（工时统计/项目统计下） 可查看项目每月的工时统计和相关详情。

2、修复用户填报历史记录，无法查看最新加入项目问题

3、一些其他优化。

# v0.5

1、修复了进入填报记录时提示的服务器错误的问题。

2、优化了填报记录里加班的计算方式。

3、优化了部分情况人员出现无需填报的场景。

4、增加了节假日的管理和导入功能。

5、优化缺报记录有时会生产多条记录的问题。

6、增加了一个windows运行的版本。详情请查看单体分支。

# V0.4

1、优化了我的工时填写页面，增加了默认工作类型和默认工时。

2、设定了compose脚本里mariadb的版本，指定为10.（11使用有问题，请谨慎使用）。

3、修订了sql脚本里因为某些主键没有自增导致的提交问题。

4、更新了前端node-sass 和sasa-loader的版本，推荐node版本为16。

5、一些其他优化。

# V0.3

1、优化了我的工时详情界面

2、增加了提交工时，选择工作类型的功能

3、增加了人员加入项目时的工时开始汇报时间：跟随项目创建时间或者加入项目时间。用于解决员工上报加入项目之前的日报。

4、一些其他优化。

# V0.2

1、新增了2023年的节假期。后台版的节日管理正在测试中

2、优化了填报了填报工时界面和操作。

3、优化了我的工时详情界面。

4、一些其他bug。

# V0.1

系统整体更新，更新内容如下：

  • 优化了初始化sql文件；

  • 优化了控制脚本调用方式；

  • 修改了前后端调用接口的方式，不在需要重新编译前端代码。现通过一个接口对外提供访问。

下载地址：工时系统-V0.1版本下载 (opens new window)

PS1:

该版本重新优化了相关数据库以及启动脚本，之前使用oak工时系统的可不进行更新。

如要更新，只主要替换后端jar和前端html文件即可。（替换之前请先做好备份，保证出错后可以恢复）。

PS2:

从该版本开始，用户不在需要重新编译前端代码即可直接使用，因此我们在程序包中添加了已编译好的前端代码，直接放到nginx或其他web服务器即可使用。
