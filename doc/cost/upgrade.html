# 系统更新

更新方式通用,请根据发布说明查看所需要的更新内容。 如版本有特殊更新则会在发布说明里说明。

# 1.普通方式部署更新

# 更新前端程序

将下载文件包里的前端程序文件，替换到原系统的前端程序文件即可。

更新前请先备份原有文件。

# 更新后端程序

1、先将后端程序停止。

2、备份原有后端程序，例如改为wuyu-cost.jar.bak。

3、将新后端程序复制到原有目录，并改名为wuyu-cost。

4、启动后端。

# 更新数据库

1、备份原数据库 登录数据库后执行导出操作

# mysql
mysqldump -uroot -p  wuyu_cost > ./wuyu_cost.sql

# mariadb
mariadb-dump  -uroot -p  wuyu_cost > ./wuyu_cost.sql

2、登录数据库并导入升级文件

# mysql
mysql -u root -p

# 进去mysql命令模式后使用
use database wuyu_cost;


# 看到提示 Database changed 后继续执行

source   /opt/updata.sql ;  #注意这里要写绝对路径。


# mariadb
mariadb  -u root -p
# 进去mariadb命令模式后使用

use database wuyu_cost;

# 看到提示 Database changed 后继续执行

source   /opt/updata.sql ; #注意这里要写绝对路径。

完成更新。

# 2.docker compose更新

1、前端程序更新 将前端文件复制到compose/conf/www 目录下即可。

（注意备份）

2、后端程序更新 将后端程序jar包复制到compose/conf/app 目录下即可。

然后将名称改为wuyu-cost.jar 并加上可执行权限

3、数据库文件更新 更新同普通方式，需要注意的是连接数据为容器中的ip 在登录命令时,需要加入-h参数,否则可能无法连接到容器里数据库 例如:

mysqldump -uroot -p -h127.0.0.1 wuyu_cost > ./wuyu_cost.sql

备注：将127.0.0.1改为mysql容器的ip 其他操作与普通方式部署更新里的数据库更新一致.
